# Trace View Core Remaining Work Plan

## 1. 문서 목적

이 문서는 현재 시점에서 실제로 남은 작업이 무엇인지,
어떤 작업이 지금 실행 가능하고 어떤 작업이 외부 입력 때문에 막혀 있는지,
그리고 다음 실행을 어떤 순서로 진행해야 하는지를 정리한다.

메인 에이전트는 이 문서를 기준으로 계획, 실행, 검증, 상태 추적을 조율한다.

## 2. 현재 기준 완료 상태

이미 완료된 범위:

- Spring 코어 분석기
- Search / latest / graph / node detail / service chain API
- Review / Approval 모델과 annotation API
- 정적 제품 레이어 UI
- 초기 empty-state 처리
- API 예외 처리 기본 보강
- 사용자 가이드 문서화
- 기본 통합 테스트 14건

근거 문서:

- `README.md`
- `docs/task-board.md`
- `docs/status-report.md`
- `docs/user-guide.md`

## 3. 남은 작업 전체 목록

### 3.1 현재 즉시 실행 가능한 작업

#### RW-001. 브라우저 레벨 UI 검증 자동화 보강

목적:

- 현재 정적 UI는 동작하지만 브라우저 상호작용 자동화 검증이 얕다.

예상 작업:

- Search -> Detail -> Graph -> Reviews 흐름 자동화
- mobile viewport 기준 레이아웃 smoke 검증
- 긴 문자열 overflow 회귀 검증

선행조건:

- 없음

우선순위:

- P1

리스크:

- 테스트 프레임워크 선택이 필요

#### RW-002. 서비스 상세 화면 분리 여부 확정 및 구현

목적:

- 현재는 node detail 안에 서비스 메서드도 포함돼 있다.
- 서비스 전용 상세가 필요하다면 별도 화면으로 분리해야 한다.

선행조건:

- 범위 확정

우선순위:

- P2

리스크:

- 현 범위에서는 node detail로도 기능상 부족하지 않을 수 있음

#### RW-003. Graph UI 고도화

목적:

- 현재 그래프는 탐색 가능하지만, 필터/레이아웃/관계 강조가 더 개선될 수 있다.

예상 작업:

- 확대/축소
- 타입별 토글
- relation 강조
- 긴 노드명 표시 전략 개선

우선순위:

- P2

## 4. 지금 기준 권장 실행 순서

### 4.1 바로 착수 가능한 순서

1. `RW-001` 브라우저 레벨 UI 검증 자동화 보강
2. `RW-002` 서비스 상세 화면 분리 여부 확정
3. `RW-003` Graph UI 고도화

### 4.2 외부 입력이 들어온 뒤 실행할 순서

1. `RW-004` 실제 외부 대상 코드베이스 연결
2. adapter 범위 확정
3. `RW-006` Browser Trace Collection
4. `RW-007` Trace Correlation

## 5. 에이전트 운영 방식

### 5.1 메인 에이전트

- 전체 우선순위와 순서 결정
- 작업 분해
- 결과 통합
- 문서 현행화

### 5.2 계획 에이전트

- 남은 작업의 실행 가능 여부 재판단
- 병렬화 가능한 작업 식별
- 작업 순서와 검증 게이트 제안

### 5.3 구현 에이전트

- 기능 단위 코드 수정
- 같은 파일 집합 충돌 방지

### 5.4 검증 에이전트

- 각 작업 완료 직후 회귀/누락 점검
- findings-first 검증 보고

### 5.5 상태 추적 에이전트

- 진행률, 병목, 다음 액션 보고
- 작업 종료 시 상태 문서 갱신 포인트 제시

## 6. 다음 실행 때의 표준 절차

1. 메인 에이전트가 현재 문서/코드 상태 재확인
2. 계획 에이전트가 남은 작업 실행 계획 제안
3. 상태 추적 에이전트가 baseline 보고
4. 구현 에이전트가 파일 충돌 없는 범위로 병렬 작업
5. 각 작업 완료 직후 검증 에이전트가 검토
6. 메인 에이전트가 수정 반영 및 재검증
7. README / status-report / task-board 현행화

## 7. 현재 결론

현재 즉시 필요한 핵심 남은 작업은 아래 3개다.

- 브라우저 레벨 UI 검증 자동화 보강
- 서비스 상세 화면 분리 여부 결정
- Graph UI 추가 고도화

현재 즉시 실행 불가능한 핵심 남은 작업은 아래 3개다.

- 실제 외부 대상 코드베이스 연결
- Browser Trace Collection
- Trace Correlation
