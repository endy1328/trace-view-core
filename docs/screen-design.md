# Trace View Core Screen Design

## 1. 문서 목적

이 문서는 Trace View Core의 주요 화면과 사용자 탐색 흐름을 정의한다.
목표는 신규 인력과 실무 개발자가 `화면 -> API -> 비즈니스 로직` 흐름을 빠르게 파악할 수 있도록 UI 구조를 고정하는 것이다.

## 2. 설계 원칙

- 검색에서 시작해 상세와 그래프로 확장되는 흐름을 기본으로 한다.
- 자동 분석 결과와 승인된 설명을 구분해 보여준다.
- 확정 관계와 추론 관계를 시각적으로 구분한다.
- 신규 인력 기준으로 학습 순서를 고려한 화면 구성을 택한다.

## 3. 사용자 플로우

### 3.1 신규 인력 플로우

1. 통합 검색에서 화면명 또는 라우트를 검색한다.
2. 화면 상세에서 호출 엔드포인트와 순서를 확인한다.
3. 엔드포인트 상세로 이동해 서버 진입점과 서비스 체인을 확인한다.
4. 그래프 화면에서 전체 호출 관계를 본다.
5. 운영 메모와 승인된 비즈니스 요약을 읽는다.

### 3.2 기능 담당 개발자 플로우

1. API 또는 서비스 메서드를 검색한다.
2. 엔드포인트 상세에서 관련 화면과 하위 호출을 확인한다.
3. 영향도 화면에서 변경 파급 범위를 점검한다.
4. 필요한 경우 주석을 추가하거나 수정 요청을 남긴다.

### 3.3 검증 담당 플로우

1. 검증 결과 화면에서 검토 대기 항목을 확인한다.
2. 근거 정보와 자동 요약 상태를 본다.
3. Must Fix와 Recommended Improvements를 등록한다.
4. 재검증 후 상태를 갱신한다.

## 4. 화면 목록

## 4.1 통합 검색 화면

목적:

- 화면명, 라우트, API, 클래스명, 메서드명을 한 번에 검색하는 진입점

주요 요소:

- 통합 검색창
- 최근 조회 목록
- 추천 온보딩 시작점
- 검색 결과 탭

주요 출력:

- 결과 유형
- 이름
- 소스 위치
- 신뢰도
- 승인 상태

## 4.2 화면 상세 화면

목적:

- 특정 화면을 기준으로 관련 API와 상위 설명을 확인

주요 요소:

- 화면 메타정보
- 라우트 경로
- 화면 컴포넌트 위치
- 호출 API 목록
- 호출 순서 타임라인
- 관련 메모

## 4.3 엔드포인트 상세 화면

목적:

- 특정 API의 진입점과 비즈니스 로직 체인을 확인

주요 요소:

- HTTP method와 path
- 호출 화면 목록
- 컨트롤러 또는 핸들러
- 서비스 체인
- 저장소 접근
- 외부 연동
- 승인된 설명과 Draft 설명

## 4.4 호출 그래프 화면

목적:

- 화면에서 서버 로직까지 연결 관계를 시각적으로 탐색

주요 요소:

- 노드 유형별 색상 구분
- 확정 관계와 추론 관계 구분
- 확대/축소
- 근거 보기 패널
- 필터링

## 4.5 서비스 상세 화면

목적:

- 특정 서비스 메서드를 기준으로 상위 엔드포인트, 하위 저장소/외부 호출, 관련 설명을 확인

주요 요소:

- 서비스 메서드 메타정보
- 상위 호출 엔드포인트 목록
- 하위 저장소 접근
- 하위 외부 연동
- 관련 근거 정보
- 승인된 설명과 Draft 설명

## 4.6 영향도 화면

목적:

- 특정 노드 변경 시 관련 상하위 노드를 확인

MVP 범위:

- 연결 노드 탐색
- 변경 위험도 점수는 후속 단계

## 4.7 주석 관리 화면

목적:

- 수동 설명, 운영 메모, 승인 상태를 관리

주요 요소:

- 주석 목록
- 작성자
- 승인자
- 상태
- 반려 사유

## 4.8 검증 결과 화면

목적:

- 문서와 분석 결과의 검증 이력과 보완 사항 확인

주요 요소:

- 검증 대상
- 검증 상태
- Must Fix
- Recommended Improvements
- 반영 여부

## 4.9 스냅샷 비교 화면

목적:

- 특정 시점과 현재 시점의 분석 결과 차이를 확인

주요 요소:

- 비교 기준 스냅샷
- 추가/삭제/변경 노드
- 관계 차이
- 영향 메모

## 4.10 운영 설정 화면

목적:

- 수집 범위, 마스킹 정책, 승인 권한, 재분석 정책 관리

## 4.11 온보딩 가이드 화면

목적:

- 신규 인력의 학습 순서와 추천 탐색 경로 제공

주요 요소:

- 1주차 추천 학습 경로
- 핵심 도메인 목록
- 필수 확인 화면
- 필수 용어집 링크

## 5. 상태 모델

### 5.1 설명 승인 상태

- Draft
- Approved
- Rejected

### 5.2 작업/문서 진행 상태

- In Progress
- Review Pending
- Needs Revision
- Validated
- Done
- On Hold
- Blocked

## 6. 화면 간 이동 규칙

- 검색 결과에서 화면 상세 또는 엔드포인트 상세로 직접 이동 가능해야 한다.
- 화면 상세에서 호출 API를 누르면 엔드포인트 상세로 이동해야 한다.
- 엔드포인트 상세에서 관련 서비스 또는 그래프로 이동 가능해야 한다.
- 엔드포인트 상세에서 서비스 상세로 이동 가능해야 한다.
- 서비스 상세에서 하위 저장소, 외부 호출, 그래프로 이동 가능해야 한다.
- 검증 결과 화면에서 대상 문서나 대상 노드로 바로 이동 가능해야 한다.
