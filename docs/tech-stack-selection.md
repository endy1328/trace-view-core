# Trace View Core Tech Stack Selection Criteria

## 1. 문서 목적

이 문서는 구현 착수 전 기술 선택 기준을 정리한다.
목표는 설계에서 구현으로 넘어갈 때 결정 공백을 줄이는 것이다.

## 2. 선택 대상

- 프론트 기술 스택
- 백엔드 기술 스택
- 정적 분석 도구
- 그래프 저장소
- 검색 인덱스
- UI 프레임워크
- 런타임 추적 연동 방식

## 3. 공통 선택 기준

- 현재 프로젝트 스택과의 적합성
- AST 또는 메타데이터 분석 가능성
- 유지보수 비용
- 학습 비용
- 도입 난이도
- 내부 배포 용이성
