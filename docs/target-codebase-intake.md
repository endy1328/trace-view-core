# Trace View Core Target Codebase Intake

## 1. 문서 목적

이 문서는 Trace View Core가 실제 대상 코드베이스에 연결되기 전에 필요한 인수 정보를 정리한다.
목표는 코드가 연결되는 즉시 구조 분석과 MVP 구현을 시작할 수 있게 하는 것이다.

## 2. 현재 상태

- 현재 작업공간에는 기획/설계/운영 문서, Spring 코어 구현 코드, 내장 참조 샘플이 존재한다.
- 내장 참조 샘플은 `Core Analyzer MVP 검증용`으로만 사용한다.
- 실제 제품 적용 대상 프론트 애플리케이션 및 백엔드 서비스 코드는 아직 연결되지 않았다.
- 따라서 개발 직전 준비와 코어 검증은 진행되었고, 실제 외부 코드 인수 단계가 다음 시작점이다.

## 3. 인수 대상

### 프론트 코드베이스

확인 항목:

- 리포지토리 경로
- 프레임워크 종류
- 라우트 정의 위치
- API 호출 wrapper 위치
- 환경 설정 방식
- 빌드 및 실행 방식

### 백엔드 코드베이스

확인 항목:

- 리포지토리 경로
- 프레임워크 종류
- 엔드포인트 선언 방식
- 서비스 계층 구조
- 저장소 계층 구조
- 외부 연동 위치
- 빌드 및 실행 방식

## 4. 필수 수집 정보

- 기본 실행 명령
- 테스트 실행 명령
- 엔드포인트 선언 규칙
- API wrapper 규칙
- 인증/인가 처리 위치
- 로그/트레이스 처리 위치
- OpenAPI 또는 API 문서 존재 여부

## 5. 구조 분석 시작 조건

아래가 확보되면 `DEV-004 Repo Structure Analysis`를 시작한다.

- 프론트 리포지토리 접근 가능
- 백엔드 리포지토리 접근 가능
- 최소 실행 명령 확인 가능
- 루트 디렉터리 구조 확인 가능

주의:

- `/samples/spring-reference-app` 분석은 위 조건을 대체하지 않는다.
- 내장 참조 샘플은 코어 검증용이며, 실제 외부 대상 코드베이스 인수와 별도다.

## 6. 인수 후 바로 수행할 작업

1. 디렉터리 구조 스캔
2. 라우트 정의 위치 식별
3. API wrapper 위치 식별
4. 백엔드 엔드포인트 선언 위치 식별
5. 서비스 계층 호출 패턴 식별
6. 저장소 및 외부 연동 위치 식별
7. 대표 화면군과 실제 코드 경로 매핑
