# Trace View Core UI Information Architecture

## 1. 문서 목적

이 문서는 Trace View Core의 UI 정보구조를 정의한다.
목표는 검색, 상세, 그래프, 운영, 검증 화면 간 정보 계층과 이동 규칙을 고정하는 것이다.

## 2. 정보 계층

### Level 1. Entry

- 통합 검색
- 온보딩 가이드

### Level 2. Exploration

- 화면 상세
- 엔드포인트 상세
- 서비스 상세
- 호출 그래프
- 영향도 탐색

### Level 3. Governance

- 주석 관리
- 검증 결과
- 스냅샷 비교
- 운영 설정

## 3. 기본 내비게이션 흐름

1. 검색
2. 상세 확인
3. 서비스 상세 확인
4. 그래프 확장
5. 근거 확인
6. 주석 또는 검증 확인

## 4. 표현 규칙

- 확정 관계와 추론 관계는 시각적으로 다르게 표시
- Draft와 Approved 설명은 분리해서 표시
- 근거가 없는 결과는 기본 뷰에 노출하지 않음
